1. Add `AnalyticsEvent` model to `prisma/schema.prisma` (append after the existing models):
   ```prisma
   model AnalyticsEvent {
     id        String   @id @default(cuid())
     eventType String   // 'PAGE_VIEW' | 'VIDEO_PLAY'
     page      String   // e.g. '/', '/projects/my-slug'
     projectId String?  // null for non-project page views
     createdAt DateTime @default(now())

     @@index([eventType, createdAt])
     @@index([projectId, eventType])
   }
   ```
   No foreign key to Project â€” analytics survives project deletion. No `updatedAt` â€” append-only.

2. Regenerate Prisma client:
   ```bash

   ```

3. Create `src/app/api/analytics/event/route.ts`:
   - Export an async `POST` function accepting `Request`
   - Parse JSON body: `{ eventType, page, projectId }`
   - Wrap the entire body in try/catch â€” analytics must NEVER error the page
   - Inside try: `await db.analyticsEvent.create({ data: { eventType: eventType ?? 'PAGE_VIEW', page: page ?? '/', projectId: projectId ?? null } })`
   - Return `new NextResponse(null, { status: 204 })` in all cases (success and error)
   - Import `NextResponse` from `next/server` and `db` from `@/lib/db`

4. Create `src/components/analytics/page-tracker.tsx`:
   - `'use client'` directive
   - Export `AnalyticsTracker` component with props: `{ page: string; eventType?: 'PAGE_VIEW' | 'VIDEO_PLAY'; projectId?: string }`
   - In `useEffect` (deps: empty array â€” fire once on mount):

     - URL: `/api/analytics/event`
     - If `navigator.sendBeacon` exists: create `new Blob([payload], { type: 'application/json' })` and call `navigator.sendBeacon(url, blob)`
     - Else: `fetch(url, { method: 'POST', body: payload, keepalive: true, headers: { 'Content-Type': 'application/json' } })`
   - Return `null` â€” component renders nothing visible
