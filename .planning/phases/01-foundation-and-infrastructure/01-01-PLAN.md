  <action>
1. Replace `src/app/globals.css` with the Tailwind v4 CSS-first theme. Do NOT create a `tailwind.config.js` — Tailwind v4 uses `@theme` in CSS:

```css
@import "tailwindcss";

@theme {
  --color-bg-base: #0A0A0A;
  --color-bg-card: #111111;
  --color-bg-section: #1A1A1A;
  --color-text-primary: #F5F5F5;
  --color-text-muted: rgba(245, 245, 245, 0.6);
  --color-accent: #FF4400;

  --font-heading: var(--font-syne), var(--font-inter), sans-serif;
  --font-body: var(--font-inter), sans-serif;
}
```

Remove any scaffold-generated default styles (the dark mode toggle styles, grid background, etc.). Keep only the `@import "tailwindcss"` and `@theme` block plus a minimal body reset.

2. Update `src/app/layout.tsx` to load Inter and Syne via `next/font/google` with CSS variables:

```tsx
import { Inter, Syne } from 'next/font/google';

const inter = Inter({ subsets: ['latin'], variable: '--font-inter', display: 'swap' });
const syne = Syne({ subsets: ['latin'], variable: '--font-syne', display: 'swap' });
```

Apply both font variables to `<html>` className. Set `<body>` to use `bg-bg-base text-text-primary font-body`. Set `lang="en"` on `<html>`.

3. Replace `src/app/page.tsx` with a design system verification page that renders:
   - A heading using `font-heading` class with the text "VLACOVISION" in large uppercase
   - A paragraph using `text-text-muted` for subtitle text
   - A div with `bg-bg-card` to verify card background
   - A span with `text-accent` to verify burnt orange accent color
   - A button with `bg-accent` and hover state

This page serves as visual proof the design system tokens work. It will be replaced in Phase 3.

4. Ensure `postcss.config.mjs` uses the Tailwind v4 plugin:
```js
const config = { plugins: { "@tailwindcss/postcss": {} } };
export default config;
```
