1. Update next.config.ts to add remotePatterns for Vimeo CDN images:
   - hostname: 'i.vimeocdn.com' (Vimeo thumbnail CDN)
   - hostname: 'vumbnail.com' (fallback thumbnail service)
   Both with protocol 'https'.

2. Create src/lib/vimeo.ts with two exported functions:
   - `getVimeoThumbnail(vimeoId: string): Promise<string | null>` — fetches from `https://vimeo.com/api/oembed.json?url=https://vimeo.com/${vimeoId}&width=1280` with `next: { revalidate: 86400 }` caching. Returns `thumbnail_url` from response. On error returns null.
   - `getProjectThumbnails(projects: Array<{ slug: string; vimeoId: string }>): Promise<Record<string, string | null>>` — calls getVimeoThumbnail for each project in parallel with Promise.all, returns slug-keyed map.

   Include a VimeoOembedData interface for the API response shape.

   Fallback: if oEmbed returns no thumbnail, try `https://vumbnail.com/${vimeoId}.jpg` as fallback URL (just return the URL string, no fetch needed — it's a CDN).
  </action>
