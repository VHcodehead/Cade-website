  <action>
Create src/components/sections/hero.tsx as a hybrid component:
- A Server Component wrapper fetches SiteConfig.heroVimeoId from db
- The inner HeroClient is 'use client' for the sound toggle interaction

**HeroClient props:** heroVimeoId: string

**Layout:**
- Full viewport height: h-screen w-full relative overflow-hidden
- Vimeo iframe as background: `https://player.vimeo.com/video/${heroVimeoId}?background=1&quality=auto`
- iframe positioned absolute inset-0, w-full h-full, pointer-events-none
- Scale the iframe slightly (scale-110) to hide Vimeo letterboxing/black bars

**Overlay:**
- Top-left: "VLACOVISION" in font-heading, text-2xl, uppercase, tracking-widest (z-10)
- Bottom-right: Sound toggle button (z-10). Icon: speaker with waves (unmuted) / speaker with X (muted). Use simple SVG icons inline.
- Sound toggle uses postMessage to Vimeo iframe: when toggling, post `{"method":"setVolume","value":0}` or `{"method":"setVolume","value":1}` to the iframe contentWindow. Give the iframe a ref. Default state: muted.
- NOTE: For postMessage to work, the iframe src must NOT use background=1 (which forces muted permanently). Instead use: `?autoplay=1&muted=1&loop=1&controls=0&title=0&byline=0&portrait=0&quality=auto`. This allows programmatic unmute via postMessage.

**Bottom gradient:**
- Absolute bottom, h-32, bg-gradient-to-t from-bg-base to-transparent

**Scroll indicator:**
- Absolute bottom-8 center, subtle animated chevron (CSS animation, simple bounce)
- Small downward arrow/chevron SVG, opacity-60, animate-bounce

**After hero (in page.tsx, not this component):**
- First CTA: "Start a Project" primary CTAButton linking to #contact
- Render this CTA in the page.tsx right after the Hero component in a centered section
