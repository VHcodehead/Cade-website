  <name>Task 1: Contact Server Action with Zod validation</name>
  <files>src/app/actions/contact.ts</files>
  <action>
Create src/app/actions/contact.ts following RESEARCH.md Pattern 6:

1. Add 'use server' directive at top

2. Define ContactSchema with Zod v4:
   - name: z.string().min(1, 'Name is required').max(100)
   - email: z.string().email('Invalid email address')
   - company: z.string().max(100).optional() (or empty string)
   - message: z.string().min(10, 'Message must be at least 10 characters').max(5000)

3. Export ContactState type:
   ```
   { status: 'idle' | 'success' | 'error'; errors?: { name?: string[]; email?: string[]; company?: string[]; message?: string[]; _form?: string[] } }
   ```

4. Export submitContact function:
   - Signature: (prevState: ContactState, formData: FormData) => Promise<ContactState>
   - Parse form data through ContactSchema.safeParse
   - If invalid: return { status: 'error', errors: parsed.error.flatten().fieldErrors }
   - If valid: call db.contactSubmission.create() with parsed data
   - On DB success: return { status: 'success' }
   - On DB error: return { status: 'error', errors: { _form: ['Something went wrong. Please try again.'] } }
   - company field: if empty string or undefined, store as empty string

Note: Check Zod v4 API — safeParse and flatten() should still work in v4. If flatten() is removed in v4, use error.issues to extract field errors manually.
  </action>

  </verify>

</task>
