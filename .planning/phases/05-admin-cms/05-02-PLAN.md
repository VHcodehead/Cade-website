<task type="auto">
  <name>Task 1: Project Server Actions</name>
  <files>src/app/actions/projects.ts</files>
  <action>
Create src/app/actions/projects.ts with 'use server' directive. Follow the established contact.ts pattern.

1. Define ProjectSchema with Zod:
   - title: string, min 1, max 200
   - client: string, min 1, max 200
   - services: string, max 500, default ''
   - year: z.coerce.number().int().min(2000).max(2099)
   - description: string, max 5000, default ''
   - vimeoId: string, min 1 (extract from full Vimeo URL or accept raw ID)
   - thumbnailUrl: string, optional (auto-fetched)
   - published: z.coerce.boolean().default(false)

2. Export ProjectActionState type:
   ```typescript
   export type ProjectActionState = {
     status: 'idle' | 'success' | 'error'
     errors?: Partial<Record<string, string[]>>
   }
   ```

3. createProject(prevState, formData): validates, generates slug from title (lowercase, replace non-alphanumeric with hyphens), auto-fetches thumbnail via getVimeoThumbnail(vimeoId) if thumbnailUrl not provided, creates with db.project.create, revalidatePath('/admin/projects') + revalidatePath('/', 'layout'), redirects to /admin/projects.

4. updateProject(id, prevState, formData): same validation, db.project.update({ where: { id } }), same revalidation, redirects to /admin/projects.

5. togglePublished(id, published): verifySession(), db.project.update({ where: { id }, data: { published } }), revalidatePath('/admin/projects'), revalidatePath('/', 'layout'). Returns void.

6. deleteProject(id): verifySession(), db.project.delete({ where: { id } }), revalidatePath('/admin/projects'), revalidatePath('/', 'layout'). Returns void.

Every action calls await verifySession() as the first line. Import verifySession from '@/lib/dal', db from '@/lib/db', getVimeoThumbnail from '@/lib/vimeo'.
