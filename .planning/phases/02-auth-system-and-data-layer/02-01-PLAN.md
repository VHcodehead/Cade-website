<task type="auto">
  <name>Task 2: Create session encryption library and Data Access Layer</name>
  <files>src/lib/session.ts, src/lib/dal.ts</files>
  <action>
1. Create src/lib/session.ts with the following exact structure (from research Pattern 1):
   - `import 'server-only'` at the very top (prevents client-side import)
   - Import `SignJWT` and `jwtVerify` from `jose`
   - Import `cookies` from `next/headers`
   - Export `SESSION_COOKIE_NAME = 'session'` as a named constant
   - Add startup guard: `if (!process.env.SESSION_SECRET) throw new Error('SESSION_SECRET environment variable is not set')` at module level
   - Encode the secret: `const encodedKey = new TextEncoder().encode(process.env.SESSION_SECRET)`
   - Define `type SessionPayload = { userId: string; expiresAt: Date }`
   - Export `async function encrypt(payload: SessionPayload): Promise<string>` — uses `new SignJWT(payload as Record<string, unknown>).setProtectedHeader({ alg: 'HS256' }).setIssuedAt().setExpirationTime('7d').sign(encodedKey)`
   - Export `async function decrypt(token: string | undefined): Promise<SessionPayload | null>` — returns null if no token, wraps `jwtVerify(token, encodedKey, { algorithms: ['HS256'] })` in try/catch returning null on failure
   - Export `async function createSession(userId: string): Promise<void>` — computes expiresAt (7 days), encrypts payload, sets cookie with httpOnly: true, secure: NODE_ENV === 'production', sameSite: 'lax', path: '/'
   - Export `async function deleteSession(): Promise<void>` — deletes the session cookie

2. Create src/lib/dal.ts with the following exact structure (from research Pattern 2):
   - `import 'server-only'` at the very top
   - Import `cache` from `react`
   - Import `cookies` from `next/headers`
   - Import `redirect` from `next/navigation`
   - Import `decrypt` from `./session`
   - Import `SESSION_COOKIE_NAME` from `./session`
   - Export `verifySession` wrapped in `cache()`:
     ```typescript
     export const verifySession = cache(async () => {
       const cookieStore = await cookies()
       const cookie = cookieStore.get(SESSION_COOKIE_NAME)?.value
       const session = await decrypt(cookie)
       if (!session?.userId) {
         redirect('/admin/login')
       }
       return { isAuth: true as const, userId: session.userId }
     })
     ```

CRITICAL NOTES:
- session.ts MUST import 'server-only' to prevent the SESSION_SECRET from leaking to client bundles
- The SESSION_COOKIE_NAME constant MUST be used in both session.ts and dal.ts (and later in proxy.ts) to prevent cookie name mismatch bugs (research Pitfall 3)
- Do NOT store email or password in the JWT payload — only userId (research anti-pattern)
- Do NOT import bcryptjs in session.ts — bcryptjs is only used in Server Actions (research anti-pattern about Edge Runtime)
